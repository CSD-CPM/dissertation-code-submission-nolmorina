Fraud Detection App

How to Run

This app runs locally on your computer or via a hosted online version. Running locally is strongly recommended — it is significantly faster and gives full control over training.

Run Locally Setup Steps Using the App CSV Columns Troubleshooting Online Version
Option 1 — Recommended

Run Locally

Running the app on your own machine gives you full speed and works after the first setup.

Extract the ZIP file. You will get a folder called Gradio with the following contents:

Gradio/
├── app.py ← the application
├── requirements.txt ← packages to install
├── artifacts/ ← pre-trained model, ready to use immediately
├── src/ ← source code
├── sample_transactions_50.csv
└── HOW_TO_RUN.md
The artifacts/ folder contains a trained model — the Fraud Scoring tab works immediately without training anything first.

Requirements: Python 3.9 or newer (check with python3 --version) and pip, which is usually installed with Python. An internet connection is needed the first time to download packages.

Setup

Step-by-Step

Follow these steps in order. You only need to install packages once.

1

Open a Terminal

Mac

Cmd + Space, type Terminal, press Enter

Windows

Windows key, type cmd or PowerShell, press Enter

Linux

Ctrl + Alt + T

2

Navigate to the Gradio Folder

cd ~/Desktop/Gradio

Confirm you are in the right place by running ls — you should see app.py listed.

3

Install Packages

Run this once. It takes 1–3 minutes — wait until the terminal prompt returns.

pip install -r requirements.txt
If pip is not found, try pip3. On Mac or Linux, add --user if you get a permissions error.
4

Start the App

python app.py
If python is not found, try python3.

You will see this output when the app is ready:

* Running on local URL:  http://127.0.0.1:7860
5

Open in Your Browser

Go to http://127.0.0.1:7860 in any browser — Chrome, Firefox, Safari, or Edge.

Keep the terminal open while you use the app. To stop it, press Ctrl + C.

Restarting later: Packages only need to be installed once. From Step 2 onwards next time, just navigate to the folder and run python app.py.

Interface

Using the App

The app has five tabs, each covering a different part of the workflow.

1
Fraud Scoring

Upload any transaction CSV. Returns a fraud probability (0–1), a binary flag, and a risk label — Low, Review, or High — sorted highest risk first. A download link for the full scored CSV is provided. The is_fraud column is not required for scoring.

2
Data Analysis

Upload a CSV to run schema validation, quality checks, and a feature engineering preview. Use this before training to spot missing columns, duplicates, or data leakage.

3
Visualizations

Generates 25 interactive charts covering fraud patterns across time, amount, geography, merchant category, and customer profile. Charts that need is_fraud show a notice when that column is absent.

4
Model Training

Upload a labelled training CSV (must include is_fraud) and optionally a holdout test CSV. Choose your model families, set an alert budget, and click Train Models. The winning model is saved automatically and becomes active in the Scoring tab.

SettingWhat it does
Model familiesWhich algorithms to train
Alert budgetTop X% of transactions to flag — e.g. 0.01 = top 1%
Max training rowsCap data size — set 0 to use all rows
Fast modeFewer trees, skips linear models — useful for a quick test run
5
Model Management

Browse all previously trained model versions and activate any of them with one click. Use this to roll back if a new training run underperforms.

Data Format

Required CSV Columns

All uploaded CSVs must contain these 13 columns. The file can be comma or pipe delimited — the separator is detected automatically.

trans_date trans_time amt category gender state dob lat long merch_lat merch_long city_pop profile
is_fraud is optional for scoring and required for training. If present during scoring it is carried through to the output unchanged.
Help

Troubleshooting

Common problems and how to resolve them.

ProblemSolution
python: command not found Use python3 instead of python
pip: command not found Use pip3 instead of pip
Port already in use Another app is using port 7860 — close it or restart your computer
ModuleNotFoundError Re-run pip install -r requirements.txt
Page not loading in browser Check the terminal still shows Running on local URL — it must stay open
Upload fails with missing columns error Check the CSV contains all 13 required columns listed above
